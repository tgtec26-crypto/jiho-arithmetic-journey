# 프로젝트 규칙 (Project Rules)

이 워크스페이스에서 작업할 때는 아래의 규칙을 반드시 준수합니다.

## 1. 폴더 생성 규칙
- 새로운 폴더를 생성할 때는 항상 **오늘 날짜**를 포함하여 **'YYMMDD_폴더명'** 형식을 사용합니다.
- 예시: `260325_지호의 산수 여행`
- 날짜 형식은 6자리 숫자(연월일)를 사용합니다.


## 3. CLI 도구 활용 및 사용자 안내
- **AI 중심 도구 활용**: `gws`, `clasp`, `vercel`, `serve`, `jq` 등의 CLI 도구는 **인공지능(AI)이 직접 사용하기 위한 도구**임을 명심합니다.
- **사용자 부담 최소화**: 사용자가 명령어를 직접 입력하거나 외울 필요가 없음을 강조하고, 모든 복잡한 옵션과 실행은 AI가 대신 수행합니다. 사용자는 설치와 로그인만 완료하면 됩니다.

## 4. 웹페이지 제작 및 배포 프로세스
- **로컬 미리보기 우선**: 웹페이지를 제작하거나 편집한 후에는 반드시 `serve` 명령어를 사용하여 로컬에서 미리 볼 수 있도록 안내한다.
- **macOS 로컬 서버 404 주의**: 제미나이 CLI 사용 시 파일명 인코딩 충돌로 404 에러가 날 수 있다. 이때는 **서버 루트(`http://localhost:포트/`)** 주소만 열어 파일 목록에서 직접 클릭하여 확인하도록 안내한다.
- **사용자 컨펌 후 배포**: 로컬 미리보기를 통해 사용자의 최종 확인(컨펌)을 받은 후에만 `vercel --prod` 명령어로 실제 배포를 진행한다.
- **자동 브라우저 열기**: 로컬 서버 실행이나 Vercel 배포 후에는 항상 `start <url>` 명령으로 브라우저에서 자동으로 열어준다.

- **[중요] 기기 최적화**: 이 프로젝트의 모든 게임은 **갤럭시탭 S6** (가로 모드, 2560x1600 해상도) 환경에 최적화되어야 한다. 스크롤 없이 한 화면에 모든 요소가 보여야 하며, 터치 기반 드래그 앤 드롭이 완벽하게 작동해야 한다.

- **[중요] 프로세스 철칙**: 수정 후에는 반드시 `npm run dev -- --host`로 로컬 미리보기를 제공하고 사용자의 확답을 받은 후 배포할 것.
